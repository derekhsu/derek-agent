# Google TypeScript Style Guide Summary

This document summarizes key rules and best practices from the Google TypeScript Style Guide.

## 1. Language Features
- Always use `const` or `let`. **`var` is forbidden.**
- Use ES6 modules with named exports.
- Avoid `any`; prefer explicit, narrow types.

## 2. Formatting and Naming
- End statements with semicolons.
- Use `UpperCamelCase` for types and `lowerCamelCase` for values.
- Do not use `_` as a prefix or suffix for identifiers.

## 3. Comments and Documentation
- Use JSDoc for documentation and `//` for implementation comments.
- Comments must add information, not restate the code.

*Source: [Google TypeScript Style Guide](https://google.github.io/styleguide/tsguide.html)*
