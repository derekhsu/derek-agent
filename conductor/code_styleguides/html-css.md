# Google HTML/CSS Style Guide Summary

This document summarizes key rules and best practices from the Google HTML/CSS Style Guide.

## 1. General Rules
- Use HTTPS for embedded resources.
- Indent by 2 spaces and use lowercase markup.
- Remove trailing whitespace and use UTF-8.

## 2. HTML
- Use valid, semantic HTML.
- Provide alt text for images.
- Separate structure, presentation, and behavior.

## 3. CSS
- Use meaningful class names.
- Prefer class selectors over ID selectors.
- Avoid `!important` unless absolutely necessary.

*Source: [Google HTML/CSS Style Guide](https://google.github.io/styleguide/htmlcssguide.html)*
